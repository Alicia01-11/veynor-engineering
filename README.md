# Veynor Engineering

Welcome to the Veynor Interactive Engineering Handbook.

This repository contains the engineering standards, architectural decisions, development workflows, templates, and operational documentation used across all Veynor Interactive software projects.

---

# Mission

Build software that is:

- Reliable
- Maintainable
- Well documented
- Secure
- Reusable
- Easy to deploy
- Easy to improve

---

# Engineering Principles

We follow these principles across every project:

1. Understand before changing.
2. Prefer small, testable changes.
3. Every meaningful change is committed to Git.
4. Security comes first.
5. Documentation is part of the product.
6. Solve a problem once, then reuse the solution.
7. Test locally before deployment.
8. Build for long-term maintenance.
